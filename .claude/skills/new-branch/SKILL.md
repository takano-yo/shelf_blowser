# new-branch — main から新規ブランチを切って作業を始める

`main` の最新を起点に新しいローカルブランチを作成し、そのブランチへ移行する。
現在どのブランチにいても、常に最新の `main` から分岐できるようにする。

## 引数

```
/new-branch <ブランチ名> [簡単な説明]
```

- `<ブランチ名>`: 作成するブランチ名（必須）。スラッシュを含む接頭辞付きでも可
  （例: `feature/login`, `fix/typo`）。
- `[簡単な説明]`: 省略可。コミットや PR 作成時の文脈として使う。

ブランチ名が渡されていない場合は、作業内容から短いケバブケース名を提案し、
ユーザーに確認してから進める。勝手に確定しない。

## 手順

実行時は次の順で進める。

1. **未コミットの変更を確認する。**
   `git status --porcelain` を実行する。変更がある場合は **そのまま勝手に破棄・
   stash しない**。ユーザーに「コミット / stash / 中止」のどれにするか確認する。
   作業ツリーがクリーンになってから次へ進む。

2. **main の最新を取得する。**
   `git fetch origin main` を実行する。
   - ネットワークエラー時は指数バックオフ（2s, 4s, 8s, 16s）で最大 4 回リトライ。

3. **最新の main を起点に新規ブランチを作成して移行する。**
   ```
   git switch -c <ブランチ名> origin/main
   ```
   - ローカルに同名ブランチが既に存在する場合は上書きしない。別名を提案して確認する。
   - `origin/main` が取得できない環境ではローカルの `main` を起点にフォールバックし、
     その旨をユーザーに伝える。

4. **結果を報告する。**
   `git status` で現在のブランチと、`origin/main` を起点にしていることを確認し、
   ブランチ名・起点・作業ツリーの状態を一言で報告する。

## 注意

- このスキルは**ブランチを作って移行するところまで**を担う。プッシュや PR 作成は
  ユーザーから明示の指示があるまで行わない。
- 既存の作業中ブランチを切り替える際に、未コミットの変更を失わせないことを最優先する。
