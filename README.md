# shelf_blowser

CiNii Books API で取得した学術書の書誌データを **本棚のように表示** し、
図書館の書架で行うようなブラウジング（請求記号順に棚を眺め、隣接する関連書に
偶然出会う体験）を Web で再現する学術検索支援アプリ。

試作版として **日本近代文学** 分野を対象とする。

---

## コンセプト

CiNii の通常検索では失われる「棚を眺めて歩く」ブラウジング体験を Web で再現する。

- **棚の並び順**: `cinii:ownerCount`（所蔵館数）の降順。所蔵が多い＝広く読まれている
  主要書ほど棚の手前（先頭）に並ぶ。
- **検索 = 棚へのワープ**: 著者・出版者・出版年などのファセットで絞り込み、
  該当する棚の位置へ移動する。

---

## データソースと役割分担

データソースは CiNii の **2 種類の API** で、**画面の機能ごとに使い分ける**。

| 機能 | 必要なデータ | データソース |
|---|---|---|
| **サイトのトップページ表示**（本棚ビュー） | タイトル・著者文字列・出版者・年・所蔵館数・シリーズ・ISBN | **OpenSearch API のみで足りる** |
| **件名・著者名を含めた詳細検索** | 件名・典拠著者（正規形）・分類（NDC/NDLC） | **図書一件ごとの詳細 API** |

トップページの本棚表示に必要なデータは **CiNii の OpenSearch から取得できるもので足りる**。
件名・著者名を含めた詳細検索を行うときに、はじめて **図書一件ごとの API の情報を利用する**。

### ① OpenSearch（一覧）— `source/日本近代文学.json`

CiNii Books OpenSearch API のレスポンス（クエリ: `日本 AND 近代 AND 文学`,
`count=10000`, `sortorder=5`）。ヒットした **5,212 件** の書誌が `@graph[0].items` に
格納されている。各 item は一覧レベルの書誌（タイトル・著者文字列・出版者・年・
所蔵館数・シリーズ・ISBN）を持つ。**トップページ表示はこの一覧情報だけで完結する。**

### ② 詳細 API（一件ごと）— `rdfs:seeAlso`

各 item は `rdfs:seeAlso` に詳細レコード（`.json`）へのリンクを持つ。
**件名 `foaf:topic`・典拠著者 `foaf:maker`（著者 ID + 正規形）・分類 `dc:subject`
（NDC/NDLC）は詳細レコード側にしか無い。** これらを使う詳細検索のために、
図書一件ごとの詳細 API を `fetch` モジュールで取得する。

## アーキテクチャ（3 モジュール構成のパイプライン）

トップページ表示用のデータ（一覧）と、詳細検索用のデータ（件名・著者名）で
取得経路が分かれる。

```
  OpenSearch（一覧）              詳細 API（一件ごと）
source/日本近代文学.json    各 item の rdfs:seeAlso
        │                            │
        │                            ▼
        │                   ┌─────────────┐  件名・典拠著者・分類の取得
        │                   │   fetch/    │  （詳細検索でのみ必要）
        │                   └─────────────┘
        │                            │
        ▼                            ▼
┌─────────────────────────────────────────┐
│   build/   検索項目の抽出・正規化・データ加工           │
│            → site/data/*.json（ownerCount 降順で整列） │
│            ・トップページ用データ … 一覧から生成        │
│            ・詳細検索用の索引（件名/著者名）… 詳細から生成 │
└─────────────────────────────────────────┘
        │
        ▼
┌─────────────┐   静的サイト（GitHub Pages 公開）
│   site/     │   本棚ビュー（一覧データ）+ 詳細検索（件名・著者名）
└─────────────┘
```

各モジュールの詳細方針は各フォルダ内の `README.md` を参照。

| モジュール | フォルダ | 役割 | 使うデータ |
|---|---|---|---|
| ① API 取得 | [`fetch/`](fetch/README.md) | 詳細書誌（件名・典拠著者・分類）を CiNii から取得 | 詳細 API。**詳細検索のために利用** |
| ② データ加工 | [`build/`](build/README.md) | 検索項目抽出・整形・索引生成 | 一覧（トップページ用）＋詳細（詳細検索用） |
| ③ サイト実装 | [`site/`](site/README.md) | 本棚 UI・ファセット検索 | トップページは一覧、詳細検索は件名・著者名 |

---

## 確定済みの要件

1. **棚の並び順** = `cinii:ownerCount` の降順
2. **トップページ表示** = CiNii の **OpenSearch（一覧）から取得できるデータで足りる**
   （`source/日本近代文学.json` の一覧情報のみで構築できる）
3. **件名・著者名を含めた詳細検索** = **図書一件ごとの詳細 API の情報を利用する**
   （件名・典拠著者・分類は詳細レコードにしか無いため）
4. **公開先** = GitHub Pages。ただし**将来的にモバイル（スマホ）表示にも
   対応できる拡張性**を持たせる
