# ② build — 検索項目の抽出・データ加工

> **ステータス: 現状 JSON（`source/日本近代文学.json`）から仮構築**
> `fetch` モジュールが未定義のため、当面は一覧レベルにある項目のみで検索項目を作る。
> 件名・典拠著者・分類は `fetch` 実装後に追加する（下記「将来追加」参照）。

---

## 目的

入力 JSON を読み込み、`site` が読む表示・検索用データへ整形する。

- 各書誌レコードを正規化したフラットな配列に変換する。
- **`cinii:ownerCount` の降順**で整列する（棚の並び順 = 主要書が先頭）。
- 検索・絞り込み用のファセット索引を生成する。

## 入力 / 出力

```
入力:  source/日本近代文学.json   （@graph[0].items に 5,212 件）
出力:  site/data/books.json       （正規化・ownerCount 降順の配列）
       site/data/facets.json      （ファセット索引）  ※構成は実装時に確定
```

## 現状で抽出可能な項目（source の一覧レベル）

`source` item のキー出現数（全 5,212 件中）:

| キー | 件数 | 抽出する項目 | 整形方針 |
|---|---:|---|---|
| `title` | 5212 | タイトル | そのまま |
| `cinii:ownerCount` | 5212 | 所蔵館数 | 数値化。**整列キー** |
| `link / @id` | 5212 | NCID・CiNii リンク | `@id` 末尾が NCID |
| `dc:publisher` | 5209 | 出版者 | 配列。ファセット化 |
| `dc:date` / `prism:publicationDate` | 5176 | 出版年 | 先頭 4 桁を年として抽出。年代（decade）も派生 |
| `dc:creator` | 4859 | 著者（表示用文字列） | 「丸山真男著」等。`著/編/校注/訳 …`等の役割語や `[ほか]` を考慮して軽く正規化・分割 |
| `dcterms:isPartOf` | 2817 | シリーズ／叢書 | `dc:title`（例「岩波新書, 青版-434」）。叢書名でファセット化 |
| `dcterms:hasPart` | 2804 | ISBN | `urn:isbn:` を除去 |

### 現状で作れる検索項目（ファセット）
- **著者**（`dc:creator` を正規化した文字列ベース。名寄せは限定的）
- **出版者**
- **出版年 / 年代**
- **シリーズ（叢書）**
- 自由語（タイトル・著者の部分一致）

> 注意: `dc:creator` は「丸山真男著」のような自由表記で、典拠 ID による厳密な
> 名寄せはできない。著者ファセットは暫定的な精度に留まる旨を UI 側にも明示する。

## 将来追加（`fetch` 実装後）

詳細レコードを入力に合流させ、以下を追加する。

- **件名ファセット** … `foaf:topic` の `dc:title`（例「日本文学 -- 歴史 -- 近代」）。
  `--` で階層分割して上位件名でも絞り込めるようにする。
- **分類ファセット** … `dc:subject` の `NDC` / `NDLC`。NDC は上位桁（類・綱・目）で
  階層ナビゲーションできるようにする。
- **典拠著者ファセット** … `foaf:maker` の著者 ID で名寄せした正規形。

## 出力レコードの想定スキーマ（暫定）

```jsonc
{
  "ncid": "BN01797559",
  "title": "日本の思想",
  "creators": ["丸山真男"],          // dc:creator を正規化
  "creatorRaw": "丸山真男著",         // 原文も保持
  "publishers": ["岩波書店"],
  "year": 1961,
  "decade": 1960,
  "ownerCount": 560,                  // 整列キー
  "series": ["岩波新書, 青版-434"],
  "isbn": ["400412039X"],
  "ciniiUrl": "https://ci.nii.ac.jp/ncid/BN01797559"
  // 将来: "subjects": [...], "ndc": [...], "ndlc": [...], "authors": [{id,name}]
}
```

## 設計方針

- 出力は GitHub Pages（静的配信）でそのまま読めるプレーン JSON とする。
- 5,212 件規模では単一 `books.json` で十分だが、件数が増えた場合や
  モバイルでの初期ロードを考慮し、**分割ロード／軽量サマリ＋詳細遅延読み込み**へ
  切り替えられる出力構造にしておく。
- 加工は冪等（同じ入力から同じ出力）にする。
