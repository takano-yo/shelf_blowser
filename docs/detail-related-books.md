# 詳細画面の関連書ナビゲーション（要件定義・実装手順）

> 本棚ページ（`site/shelf.html`）の書誌詳細オーバーレイに **関連書（同じ著者・同じ出版社・同じ NDC）** を
> 表示し、詳細画面のまま関連書へ遷移できるようにする。あわせて **NDC 棚への逆引きリンク** と
> **オーバーレイの拡幅（PC で画面半分程度・1 画面収まり）** を行う。
> 優先度はロードマップ P2（サイト構成）の拡張。ページ構成・NDC 棚データの前提は
> [docs/site-structure.md](site-structure.md) に従う。

---

## 背景と現状

- 棚ページでは表紙カードをクリックすると詳細オーバーレイ（`#overlay`・画面右の
  幅 `min(28rem, 92vw)` パネル）が開き、書誌情報（出版社・出版年・所蔵館数・ISBN・
  著者表記・NCID）・シリーズ情報・CiNii リンクを表示する（`site/js/app.js` の
  `overlayHtml` / `openOverlay`）。
- 詳細から他の書誌への横の移動手段がなく、行き止まりになっている。
- 書誌レコード（`books.json` スキーマ）は **NDC 分類を持たない**。NDC 分類棚は
  `site/data/ndc/<記号>.json`（1〜3 桁・各分類とも所蔵館数上位 1,000 件収録）として
  静的に持っている。

### データの実測（2026-07-22・リポジトリ内の実データ）

| 項目 | 実測値 |
|---|---|
| 3 桁分類棚ファイル | 1,000 ファイル・延べ 826,548 レコード |
| ユニーク NCID | 774,473 件（うち複数の 3 桁分類に載るもの 49,032 件 ≒ 6.3%） |
| 既定棚（`books.json` 5,212 件）のうち 3 桁分類棚に載っている書誌 | 785 件（15.1%） |
| 1 桁棚（例 `9.json` 1,000 件）のうち 3 桁分類棚に載っている書誌 | 1,000 件（100%） |

> 1・2 桁棚の書誌が 3 桁棚に必ず載るのは、部分集合内の所蔵館数順位は全体での順位以上に
> なるため（上位 1,000 件切り詰めの単調性）。既定棚は検索語ベースのデータセットなので
> 大半（約 85%）が分類棚未収録＝**NDC 不明**になる。この制約は仕様として受け入れる（後述）。

## 要件

### R1. 関連書の提示

- 詳細画面の書誌情報・CiNii リンクの**下**に関連書エリアを設ける。
- **「同じ著者」「同じ出版社」「同じ分類（NDC）」の 3 種**を、それぞれ**最大 10 件**提示する。
- 一致条件はすべて**完全一致**:
  - 著者: 表示中書誌の `creators` と候補の `creators` に、同一文字列の著者名が 1 つ以上ある。
  - 出版社: 同様に `publishers` 同士に同一文字列が 1 つ以上ある。
  - NDC: **3 桁**分類の一致。書誌の 3 桁 NDC は **3 桁分類棚の元ファイル
    （`site/data/ndc/<3桁>.json`）に載っているかどうか**で判定する（レコード自体は
    NDC を持たないため。判定用の逆引きインデックスを D1 で新設する）。
- 候補の母集合:
  - 著者・出版社: **現在棚に表示中のデータセット全体**（既定棚 `books.json`／NDC 棚
    `ndc/<記号>.json`／検索結果。タブ・棚内検索で絞り込む前の全体）。
  - NDC: 表示中書誌の 3 桁 NDC の**分類棚ファイル `ndc/<3桁>.json`**（オンデマンド取得）。
- 並びは**所蔵館数（ownerCount）降順**で上位 10 件。
- 除外: 表示中の書誌自身（同一 NCID）と、表示中書誌と**同じシリーズ親**（`series[0].id`
  が一致）の巻。シリーズの巻は既に詳細内の「シリーズ収録巻リスト」で提示しており、
  関連書行が同シリーズの巻で埋まるのを防ぐ。
- 0 件の種別は行ごと非表示にする（見出しだけ残さない）。NDC 不明の書誌は NDC 行を出さない。

### R2. 関連書の表示形式

- 関連書は**本棚と同じ表紙（書影 or プレースホルダー）のみ**で表示する
  （表紙下のタイトル・著者などのメタ行は付けない）。
- 3 種をそれぞれ**横 1 列**に並べ、収まらない場合は**横スクロール**（`overflow-x: auto`）。
- 各行に小さな見出しを付ける（「同じ著者」「同じ出版社」「同じ分類 NDC 913 小説.物語」の類。
  NDC の分類名は `data/ndc/index.json` から引く）。

### R3. 詳細画面内の遷移（履歴つき）

- 関連書の表紙クリックで、**オーバーレイを開いたまま**その書誌の詳細へ差し替える
  （何度でも繰り返し可能）。遷移先でも関連書を再計算して表示する。
- パネル**左上に「戻る（<）」「進む（>）」ボタン**を設け、オーバーレイ内の閲覧履歴を
  行き来できる。それ以上戻れない／進めない側は無効表示（disabled）。
- 履歴は**オーバーレイ内で完結**させ、ブラウザの履歴・URL には反映しない
  （既存の `?q=/?ndc=/tab/sort` の URL 状態同期を汚さない）。オーバーレイを閉じたら破棄。
- 関連書から開いた書誌は**単独書誌として**表示する（棚でシリーズ束に属する巻でも、
  束の代表ではなくその巻自体の詳細を出す。シリーズ情報は `series` フィールドから表示）。

### R4. NDC 棚への逆引きリンク

- 詳細画面に、表示中書誌の NDC の **1 桁・2 桁・3 桁の各棚へのリンクボタン**を並べる
  （例: NDC 913 → `[9 文学] [91 日本文学] [913 小説.物語]`。ラベルは `index.json` から）。
- リンク先は `shelf.html?ndc=<記号>`（通常のページ遷移。背景の棚ごと切り替わる）。
- **現在の背景の棚（`?ndc=` の値）と一致する記号のボタンはグレーアウト**（無効化）する。
  既定棚・検索結果棚（`?ndc=` なし）ではどれもグレーアウトしない。
- 書誌が複数の 3 桁分類に載る場合（約 6%）は**主分類 1 つ**に絞る: 背景の棚の NDC と
  前方一致するコードがあればそれを優先し、なければ記号昇順の先頭を使う。
- NDC 不明の書誌ではボタン群を表示しない。

### R5. レイアウト拡幅（1 画面収まり）

- PC（幅 600px 超）でオーバーレイパネルを**画面の半分程度**（`width: min(max(28rem, 50vw), 92vw)` 目安）
  に拡幅する。
- 標準的な PC 画面（フル HD・ウィンドウ高さ 900px 程度）で、**書誌詳細＋関連書 3 行が
  縦スクロールなしで 1 画面に収まる**ことを目標にレイアウトを調整する:
  - 上段を**書影（左）＋書誌情報（右）の 2 カラム**にして縦を圧縮する。
  - シリーズ収録巻リスト（巻数が多いと長い）は**リスト内スクロール**（max-height）にする。
  - 関連書の表紙は棚より小さいサイズ（幅 5〜6rem 目安）で 1 行あたりの高さを抑える。
- モバイル（600px 以下）は現行のボトムシート表示を踏襲し、縦スクロールは許容する
  （関連書行の横スクロールはモバイルでも機能する）。
- 縦が収まらない環境（低い解像度・大きいフォント設定など）では従来どおり
  パネル内の縦スクロールにフォールバックする（`overflow-y: auto` は維持）。

## データ設計

### D1. NCID → 3 桁 NDC の逆引きインデックス（新設）

書誌レコードに NDC を持たせる代わりに、**コミット済みの 3 桁分類棚ファイルから導出する
逆引きインデックス**を静的データとして追加する。`books.json` スキーマ（唯一の契約）は
**不変**のまま、既存データファイルにも一切手を入れない。

```
site/data/ndc/rev/
├── index.json   # メタ: 生成日時・シャード方式・件数（sourceFiles/records/uniqueNcids）
└── <XX>.json    # シャード本体。XX = NCID の末尾 2 文字（例 91.json, 4X.json）
                 # 中身: { "<NCID>": ["913"], "<NCID>": ["000","031"], ... }（キー昇順）
```

- **シャード方式**: NCID 末尾 2 文字。実測で 110 シャード・各約 7,000 件
  （最小 6,868／中央 7,036／最大 7,291）とほぼ均等。1 シャード約 200KB
  （gzip 配信で数十 KB）・全体約 20MB。クライアントは詳細を開いた書誌の NCID から
  該当シャード 1 個だけを取得しキャッシュする。
- **生成**: `build/build.py --ndc-rev` を新設。`site/data/ndc/` の **3 桁ファイルのみ**を
  走査して生成する（`--ndc-covers-inplace` と同じ「コミット済み成果物から導出」方式。
  `.cache/ndc/` の生キャッシュは不要）。キー昇順・値昇順で出力し**冪等**にする。
- **更新運用**: NDC 棚データを再生成・更新したら `--ndc-rev` も再実行する
  （ランブックに追記）。値が変わらなければ再実行で差分ゼロ（`index.json` の
  生成日時のみ更新）。
- 複数分類に載る NCID は値が複数コードの配列になる（昇順）。

### D2. 取得とキャッシュ（クライアント）

- 逆引きシャード・NDC 分類棚ファイル・`ndc/index.json` は、それぞれ **Promise を
  Map にキャッシュ**して同一セッション中の再取得を避ける（`loadNdcIndex` と同方式）。
- 逆引きシャードの取得失敗（404 等）は「NDC 不明」と同じ扱いに落とす
  （NDC 行・逆引きボタンを出さないだけで、詳細表示自体は成立させる）。

## 判断事項（要件の曖昧さの解消）

実装前に確定した判断。いずれも変更容易な箇所なので、意図と違えば後続 PR で調整する。

| # | 論点 | 判断 | 理由 |
|---|---|---|---|
| 1 | 書誌の NDC の求め方 | 3 桁分類棚ファイル収録の有無から逆引き（D1）。レコードへの `ndc` フィールド埋め込みはしない | スキーマ契約・既存データ約 400MB を不変に保つ。埋め込みは全データ書き換え（Git 履歴の肥大）になる |
| 2 | 著者・出版社の候補母集合 | 現在棚のデータセット全体 | 静的サイトで全書誌横断の完全一致検索は不可能。棚の文脈に沿った「この棚の中の関連書」として自然 |
| 3 | 複数著者・複数出版社の扱い | いずれか 1 つの完全一致で「同じ」とみなす | 「同じ著者の本」の直感に合う。第 1 著者限定にすると共著の取りこぼしが大きい |
| 4 | 複数 NDC を持つ書誌 | 主分類 1 つに絞る（背景の棚と前方一致優先→記号昇順先頭） | 行・ボタン群の増殖を防ぐ。該当は約 6% |
| 5 | 同一シリーズの巻の除外 | 関連書行から除外 | シリーズ巻は既に詳細内のリストで提示済み。著者行が同シリーズで埋まるのを防ぐ |
| 6 | 履歴と URL | オーバーレイ内履歴はブラウザ履歴と独立 | 既存の URL 状態同期（?q/?ndc/tab/sort）と衝突させない。専用の戻る/進むボタンを要件どおり設ける |
| 7 | 動的検索結果の書誌 | 逆引きは NCID ベースなのでそのまま機能する。著者・出版社行の母集合は検索結果セット | 追加実装なしで一貫した挙動になる |
| 8 | 0 件・NDC 不明時 | 該当行・ボタン群を非表示 | 空の見出しやプレースホルダーはノイズ。既定棚では NDC 不明が約 85% ある（実測）ため、出せる場合のみ出す |

## 制約・既知の限界（仕様として受け入れる）

- 関連書は**リポジトリ収録データの範囲**（各分類上位 1,000 件・既定棚 5,212 件）に限られる。
  全 CiNii 横断の関連書ではない。
- 既定棚（日本近代文学）の書誌は約 15% しか 3 桁分類棚に載っておらず、NDC 行・
  逆引きボタンが出ない書誌が多い。NDC 棚（?ndc=）経由では 1〜3 桁とも実質 100% 出る。
- 逆引きインデックスは生成時点の分類棚のスナップショット。棚データ更新時は再生成が必要。
- 動的検索（`/api/search`）の結果に対する著者・出版社行は、その検索結果内での一致に限る。

## 実装手順（auto-ship 単位）

各手順を独立の PR として作成し、整合性確認のうえ auto-ship（自動マージ）する。
いずれも静的サイト＋派生データの追加であり、既存データファイルの書き換え・削除、
依存関係・CI・スキーマ契約の変更を含まない。

| 手順 | 内容 | 主な変更 | 検証 |
|---|---|---|---|
| 1 | **本書（要件定義）の追加** | `docs/detail-related-books.md` | リンク・参照の整合 |
| 2 | **逆引きインデックス生成**: `build.py --ndc-rev` 新設＋ `site/data/ndc/rev/` 生成・コミット（新規約 110 ファイル・約 20MB。既存ファイル無変更） | build / site/data | 生成の冪等性・件数一致（総 NCID 774,473）・JSON 妥当性・既存ファイル無変更をスクリプトで確認 |
| 3 | **オーバーレイ拡幅**: PC で約 50vw・書影＋書誌の 2 カラム・シリーズ巻リストの内部スクロール | site (css/js/html) | ローカル配信＋Playwright で PC/モバイル表示・開閉・既存操作の回帰確認 |
| 4 | **関連書 3 行＋オーバーレイ内遷移＋戻る/進む** | site (js/css) | 同上＋一致条件・除外・10 件上限・横スクロール・履歴動作の確認 |
| 5 | **NDC 逆引きリンクボタン**（1/2/3 桁・背景一致でグレーアウト） | site (js/css) | 同上＋NDC 棚背景での グレーアウト・遷移確認。1 画面収まりの最終調整 |

- 手順 2 は差分こそ大きいが**機械生成データの新規追加のみ**（既存ファイル不変・
  当面どのコードからも未参照）で、失注時は `rev/` を消すだけで戻せるため auto-ship の
  対象とする。検証ゲート（冪等性・件数・妥当性）を必須とする。
- 手順 3〜5 は `site/` 配下のみの変更。マージごとに Pages が自動デプロイされる
  （`pages.yml` は `site/**` パスで発火）。CSS/JS のキャッシュバスター（`?v=`）を
  手順ごとに更新する。

## 受け入れ条件

1. 詳細画面の書誌情報・CiNii リンクの下に、同じ著者・同じ出版社・同じ NDC（3 桁）の
   関連書が**表紙のみ・横 1 列・最大 10 件ずつ**表示され、あふれた分は横スクロールできる。
2. 一致はすべて完全一致で、表示中書誌自身と同一シリーズ親の巻は関連書に含まれない。
3. 関連書クリックで詳細画面のままその書誌へ遷移でき、繰り返し遷移できる。左上の
   「<」「>」ボタンでオーバーレイ内履歴を行き来でき、端では無効表示になる。
4. 詳細画面に表示中書誌の NDC 1・2・3 桁棚へのリンクボタンが並び、背景の棚と一致する
   記号のボタンはグレーアウトされる。NDC 不明の書誌では表示されない。
5. PC でオーバーレイが画面の約半分に拡幅され、フル HD 級の画面で書誌詳細＋関連書が
   縦スクロールなしで収まる。モバイルは現行のボトムシート挙動を維持する。
6. `site/data/ndc/rev/` の再生成が冪等で、既存の `books.json`・`ndc/*.json` は
   バイト単位で無変更のまま残る。
