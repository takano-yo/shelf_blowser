# ③ site — 表示サイト（本棚 UI ＋ファセット検索）

> **ステータス: 現状 JSON から仮構築**
> `build` が出力する `site/data/*.json` を読み込んで表示する。
> 公開先は GitHub Pages。将来のモバイル対応を見据えた構成にする。
>
> 本書は**要件定義**を主とする。配色・タイポグラフィ等の視覚デザインの詳細は
> 別途定義し、ここでは「何を・どう振る舞うか」を規定する。

---

## 目的

書誌データを **本（表紙）を横に並べた本棚** として表示し、図書館の書架
ブラウジングを Web で再現する。

- 各書誌を **`cinii:ownerCount` 降順** で棚に並べる（所蔵館が多い＝主要書ほど手前）。
- ウィンドウ幅に合わせて折り返し、棚（行）が下へ続く。
- 書影クリックで書誌詳細を表示する。

## 背表紙ではなく「表紙」を見せる（方針変更）

旧版は背表紙を縦に並べる構成だったが、本サイトは **表紙（カバー面）を並べる**
構成に変更する。表紙画像は ISBN を用いて外部 API から取得し、取得できない場合は
**プレースホルダー**を表示する（後述）。

## データの使い分け（トップページ vs 詳細検索）

- **トップページ表示**（本棚ビュー）… CiNii の **OpenSearch（一覧）から取得できる
  データだけで足りる**。`build` が一覧から生成した `site/data/books.json` を読み込む。
- **件名・著者名を含めた詳細検索 … 現フェーズでは対象外（後回し）。**
  件名・典拠著者・分類は `fetch`（一件ごとの詳細 API）が必要なため、当面は実装せず、
  詳細表示も **`books.json`（一覧データ）内の項目のみで代替**する。

## 技術構成

- **静的サイト + バニラ JS**（ビルド工程なし）。HTML / CSS / JS のみ。
- `build` が出力した `site/data/*.json` を `fetch()` で読み込み、表示・整列は
  **クライアントサイド**で行う。
- GitHub Pages にそのまま配置して公開できる（サーバ不要）。

### なぜこの構成か
- 試作に最適（環境構築・ビルド不要、すぐ公開できる）。
- データが静的なので、表示・整列がクライアント側で完結する。
- 将来フレームワーク化する場合も、データ層（`site/data/*.json`）は再利用できる。

## 画面構成

### 1. 本棚ビュー（メイン）

書影を横一列に並べ、ウィンドウ幅で折り返して複数段の「棚」を構成する。

- **並び順**: `ownerCount` 降順。
- **レイアウト**: ウィンドウ幅に応じて 1 行に並ぶ冊数を変え、行が下へ続く。
  書影は **一定のアスペクト比（書籍標準＝およそ 1 : 1.45）に固定**し、棚として
  整列させる。実画像は枠に合わせてトリミング表示（`object-fit: cover` 等）する。
- **書影の下に表示する項目**: タイトル / 著者名 / 出版社名 / 出版年。
- **ホバー**: 書影にマウスを重ねると、より詳細な書誌情報を表示する
  （タイトル・著者・出版社・出版年・所蔵館数・シリーズ・ISBN・CiNii リンク等、
  `books.json` にある項目）。
- **クリック**: 各書影は詳細画面へのリンクとして機能する（後述の詳細オーバーレイ）。

#### 表紙画像とプレースホルダー

- 表紙画像は **ISBN を用いて外部 API から取得**する。取得は**ビルド時にまとめて
  行い**（クライアントから個別に大量取得しない＝API へのマナー／性能上の理由）、
  結果を `books.json` の各レコードに `coverUrl`（無ければ `null`）として埋め込む。
- **取得は複数ソースのフォールバック**で行う。例: `OpenBD → NDL サーチ →
  プレースホルダー`。本データは複刻・全集・古い学術書が中心で書影カバー率が低く、
  **多くがプレースホルダーになる前提**で設計する（実データでは ISBN 保有が約 54%、
  かつ書影が存在する割合も低い）。
- **プレースホルダーの表示内容**:
  - 画像の**中央〜上**あたりに **タイトル・著者名**。
  - 画像の**下辺**に **出版社名**（シリーズものの場合はシリーズ名に置き換える）。

#### シリーズ（親書誌）のまとめ表示

`dcterms:isPartOf`（親書誌）を持つ書誌は、**同一の親書誌 ID 単位でまとめる**。

- **代表表示**: そのシリーズに属する中で **`ownerCount` が最大の 1 冊**の表紙を
  代表として表示する。
- **並び順の位置**: シリーズの束は、**シリーズ内の最大 `ownerCount`** を基準に
  本棚へ配置する（合算しない）。
- **「本が連なる」表現**: 代表表紙の背後に紙（本）が重なって続くイメージの影を
  付ける（`]]]]` のように右後ろへ重なる影）。
- **シリーズの明示（枠は 2 値）**: シリーズものは**枠あり**、単独本は**枠なし**で
  区別する（シリーズごとの固有色分けは行わない＝親書誌は 794 件と多く識別不能なため）。
- **下部の表示**: 書影の下の **出版社名の代わりにシリーズ名**を表示する。

### 2. 検索窓（現フェーズはダミー）

- 画面に検索窓を **配置するが、現フェーズでは機能しないダミー**とする。
- 将来的にファセット検索（出版者 / 出版年・年代 / シリーズ、著者・自由語の部分一致）へ
  拡張する。件名・典拠著者・分類による詳細検索は `fetch`／`build` 整備後に追加する。

### 3. 書誌詳細（オーバーレイ）

- 書影クリックで **画面右にオーバーレイ表示**する（モバイル版の出し方は別途検討）。
- **現フェーズの表示項目**: `books.json`（一覧データ）にある項目のみ
  ── タイトル・著者・出版社・出版年・所蔵館数・シリーズ・ISBN・**CiNii へのリンク**。
- 分類・件名は詳細 API（`fetch`）が必要なため**後回し**。将来、`build` が生成する
  詳細索引が用意でき次第、本オーバーレイに追加する。

## モバイル対応の方針（将来の拡張性）

GitHub Pages 公開時点では PC 表示が主だが、後からスマホ表示へ拡張できるよう
最初から以下を守る。

- **レスポンシブ設計**: メディアクエリで画面幅に応じて 1 行の冊数・余白を切り替える。
  狭幅では列数を減らし、書影サイズを縮める。
- **相対単位**: `rem` / `%` / `vw` 等を用い、固定 px レイアウトを避ける。
- **タッチ操作**: クリックだけでなくタップを想定（hover 依存の情報は、タップでも
  到達できる導線を別途用意する）。
- **viewport メタ**: `<meta name="viewport">` を最初から設定。
- **詳細オーバーレイ**: PC では画面右に表示。モバイルでの出し方（全画面／ボトム
  シート等）は別途検討する。
- **パフォーマンス**: 大量の書影描画はモバイルで重くなるため、仮想スクロール／
  遅延描画へ差し替えられる構造にしておく（`build` の分割出力と対）。

  表紙画像 URL（`coverUrl`／無ければ `null`）を含める。
