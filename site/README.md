# ③ site — 表示サイト（本棚 UI ＋ファセット検索）

> **ステータス: 現状 JSON から仮構築**
> `build` が出力する `site/data/*.json` を読み込んで表示する。
> 公開先は GitHub Pages。将来のモバイル対応を見据えた構成にする。

---

## 目的

書誌データを **本棚のように表示** し、図書館の書架ブラウジングを Web で再現する。

- 背表紙を **`cinii:ownerCount` 降順** で棚に並べる。
- ファセット検索で絞り込み、該当する棚の位置へ移動する。
- 背表紙クリックで書誌詳細（CiNii へのリンク含む）を表示する。

## データの使い分け（トップページ vs 詳細検索）

- **トップページ表示**（本棚ビュー）… CiNii の **OpenSearch（一覧）から取得できる
  データだけで足りる**。`build` が一覧から生成した `site/data/books.json` を読み込む。
- **件名・著者名を含めた詳細検索** … 件名・典拠著者・分類は一覧に無いため、
  `fetch` が取得した **図書一件ごとの API の情報**（`build` が詳細検索用に出力した
  索引）を利用する。

## 技術構成

- **静的サイト + バニラ JS**（ビルド工程なし）。HTML / CSS / JS のみ。
- `build` が出力した `site/data/*.json` を `fetch()` で読み込み、
  検索・絞り込みは**クライアントサイド**で行う。
- GitHub Pages にそのまま配置して公開できる（サーバ不要）。

### なぜこの構成か
- 試作に最適（環境構築・ビルド不要、すぐ公開できる）。
- データが静的なので、クライアント側の索引で検索が完結する。
- 将来フレームワーク化する場合も、データ層（`site/data/*.json`）は再利用できる。

## 画面構成

1. **本棚ビュー（メイン）**
   - 背表紙を `ownerCount` 降順で横一列に並べ、折り返して複数段の「棚」を構成。
   - 背表紙の見た目（高さ・厚み・色）にメタデータを反映:
     - 厚み・高さ → 所蔵館数や出版年などにマッピング（視覚的な手がかり）。
     - 色 → 年代やシリーズなどで色分け（検討）。
   - 背表紙にタイトル・著者を縦書き風に表示。
2. **ファセット検索パネル**
   - **一覧データで完結する絞り込み**: 出版者 / 出版年（年代）/ シリーズ、
     著者（一覧の文字列ベース、暫定精度）、自由語（タイトル・著者の部分一致）。
   - **詳細検索（一件ごとの API 情報を利用）**: **件名**・**著者名（典拠の正規形）**・
     分類（NDC/NDLC）で絞り込み。これらは `fetch` 由来の詳細データを使う。
   - 絞り込み結果は本棚側にハイライト＋件数表示し、該当位置へスクロール。
3. **書誌詳細**
   - 背表紙クリックで詳細パネル／モーダル表示。
   - タイトル・著者・出版者・年・所蔵館数・シリーズ・ISBN・**CiNii へのリンク**。

## モバイル対応の方針（将来の拡張性）

GitHub Pages 公開時点では PC 表示が主だが、後からスマホ表示へ拡張できるよう
最初から以下を守る。

- **レスポンシブ設計**: メディアクエリで画面幅に応じてレイアウトを切り替える。
  - 棚は横スクロール（タッチでスワイプ）を基本とし、狭幅では段組み／カードへ縮退。
  - ファセットパネルは PC ではサイド固定、狭幅では引き出し（ドロワー）化。
- **相対単位**: `rem` / `%` / `vw` 等を用い、固定 px レイアウトを避ける。
- **タッチ操作**: クリックだけでなくタップ・スワイプを想定（hover 依存を避ける）。
- **viewport メタ**: `<meta name="viewport">` を最初から設定。
- **パフォーマンス**: 大量の背表紙描画はモバイルで重くなるため、
  仮想スクロール／遅延描画へ差し替えられる構造にしておく（`build` の分割出力と対）。

## 想定ファイル構成（暫定）

```
site/
├── README.md
├── index.html
├── css/styles.css
├── js/app.js          # データ読込・本棚描画・ファセット検索
└── data/              # build の出力（books.json など）
```

## 公開（GitHub Pages）

- `site/` 配下を Pages の公開ルートにする（または `docs/` 運用へ調整）。
- すべて相対パスで参照し、サブパス公開（`/<repo>/`）でも動くようにする。

## 入力データ

- **トップページ用**: `build` の出力 `site/data/books.json`（`ownerCount` 降順）。
  OpenSearch（一覧）から生成され、本棚表示はこれだけで足りる。
- **詳細検索用**: 件名・著者名・分類の索引（`build` が一件ごとの詳細 API から生成）。
  件名・著者名を含めた詳細検索を有効化する場合に読み込む。

スキーマは `build/README.md` を参照。
