# ③ site — 表示サイト（スタートページ＋本棚 UI）

> **ステータス: スタートページ（入口）＋本棚ページの 2 ページ構成で実装済み・
> 公開中（GitHub Pages）。**
> 本棚ページは `build` が出力する `site/data/books.json` を初期表示する。検索は
> **収録データ検索（既定・サーバ不要。事前生成コーパスから NDC 類内を検索）**と
> **CiNii API 検索（`server/app.py` の `/api/search`・稼働時のみ）**の 2 モード。
> いずれも `books.json` と**同一スキーマ**の books 配列で本棚を作り直す。
> バニラ JS・ビルド工程なし。
>
> 本書は「何が実装済みか（現状）」と「今後必要な作業（要件・手順）」を規定する。

---

## 目的

書誌データを **本（表紙）を横に並べた本棚** として表示し、図書館の書架
ブラウジング（棚を眺めて歩き、隣接する関連書に偶然出会う体験）を Web で再現する。

## ページ構成

サイトは **スタートページ（入口）→ トップページ（本棚ページ）** の 2 ページ構成
（2026-07-14 実装）。全体要件は [docs/site-structure.md](../docs/site-structure.md) を正とする。

| ページ | ファイル | 役割 |
|---|---|---|
| スタートページ | `index.html` | 検索窓＋NDC 分類ボタン（類目→綱目→細目）で初期条件を指定し本棚ページへ遷移 |
| 本棚ページ | `shelf.html`（旧 `index.html` を移設） | `?q=<語>` / `?ndc=<分類記号>` を受け取り棚を表示。クエリなしは既定データ（後方互換） |

## 技術構成

- **静的サイト + バニラ JS**（ビルド工程なし）。HTML / CSS / JS のみ。
- `site/data/*.json` を `fetch()` で読み込み、表示・整列は**クライアントサイド**で行う。
- GitHub Pages にそのまま配置して公開できる（初期表示はサーバ不要）。
- 動的検索はバックエンド（`server/app.py`）の `/api/search` を利用する。
  API が無い・落ちている場合も初期の本棚は静的データで表示される
  （グレースフルデグレード）。

```
site/
├── README.md
├── index.html         # スタートページ（検索窓＋NDC 分類ナビ・出典表示）
├── shelf.html         # 本棚ページ（旧 index.html。?q= / ?ndc= を受け取る）
├── css/styles.css
├── js/app.js          # 本棚ページ: データ読込・描画・タブ/並べ替え・検索・URL 状態同期
├── js/search.js       # 収録データ検索（正規化・コーパス走査・書誌解決・/api/ping・AbortController）
├── js/start.js        # スタートページ: NDC 分類ナビ（3 階層）・検索モード切替の描画
└── data/              # build の出力（books.json / meta.json）
    ├── ndc/           # NDC 分類ごとの棚データ＋マスタ index.json
    └── search/        # 検索コーパス（Pages デプロイ時生成・Git 管理外・.gitignore）
```

## 実装済みの機能（現状）

### スタートページ（入口・`index.html`）

- **検索窓＋モード切替**: 「収録データ検索（既定・サーバ不要）」と「CiNii API 検索
  （サーバ稼働時のみ）」を切り替える。収録データ検索では類（0〜9・分類名つき）を
  選んで `shelf.html?ndc=<類>&q=<語>` へ、API 検索では `shelf.html?q=<語>&mode=api`
  へ遷移する（検索の実行は本棚ページ側。スタートページは `/api/ping` で稼働判定
  のみ行い、未稼働時は API 検索をグレーアウトする）。JS 無効時は素の GET フォーム
  として `?q=` のみを送る（後方互換）。
- **NDC 分類ナビ**: 類目（1 桁）→ 綱目（2 桁）→ 細目（3 桁）を各階層 10 個の
  ボタンで掘り下げる（`js/start.js` が NDC マスタ `data/ndc/index.json` から
  記号・分類名・件数を描画）。「上の階層へ戻る」と、1・2 桁時の
  「この分類の棚を見る」（`shelf.html?ndc=<記号>` へ遷移）を備え、3 桁は選択＝遷移。
  データ未整備・0 件の分類は無効表示。NDC9 の欠番（分類名なし・60 分類）は
  「分類名未収録」と表示する。
- **出典表示**: 分類名（JLA 公式 NDC 新訂9版データ〈NDC-LD〉・CC BY）・
  書誌データ（CiNii Books・CC BY 4.0）・書影（openBD）の提供元・ライセンス・
  リンクをページ内に明記。
- 本棚ページのタイトル（ロゴ）がスタートページへ戻るリンクになっている。

### 本棚ビュー

- **並び順**: `ownerCount`（所蔵館数）降順。主要書ほど棚の手前に並ぶ。
- ウィンドウ幅から列数を算出して明示グリッドに固定（最低 3 列）し、
  段ごとに「棚板」を描画する。
- 書影は書籍標準のアスペクト比で整列（実画像は `object-fit: cover`）。
- **プレースホルダー**（書影の無い書誌＝大多数・約 96%）: タイトル・著者を上部、
  出版社（シリーズものはシリーズ名）を下辺に描画。地色と下辺の文字色は
  **出版年に応じた連続グラデーション**（古＝暗い赤 → 中＝緑 → 新＝水色）で塗り、
  棚を一望したとき年代の分布が見えるようにしている。
- **無限スクロール**: 先頭 100 件（`PAGE_SIZE`）のみ描画し、棚末尾のセンチネルが
  近づくたびに次の 100 件を追加描画する。
- **ホバー詳細**: 表紙にマウスを重ねると、表紙と同サイズの詳細
  （著者・出版社・出版年・所蔵館数・シリーズ・ISBN）へフェードで交代する。

### シリーズ（親書誌）のまとめ

- `series[0].id`（親書誌 NCID）単位で束ね、**所蔵館数最大の 1 冊を代表表示**
  （棚での位置基準も代表の所蔵館数）。
- 「本が連なる」紙束の影・枠・冊数バッジで単独本と区別する。
- 多巻単独（ISBN 数 ≥ 4）も「シリーズ的」として同じ装飾の対象にする。

### タブ・並べ替え・シリーズまとめ解除

- **タブ**: すべて／単著・共著／編集書／シリーズ。`contribKind`（build 付与）と
  シリーズ的か否かで 3 バケットに分け、各タブは該当書籍のみを所蔵館数順に表示。
  「すべて」タブは**ブロック配分**（4〜8 冊のブロックへ編集書 1 冊・シリーズ 1 冊を
  混ぜる。固定シード乱数で並びは再現可能）。タブ間でスクロール位置を保存・復元する。
- **並べ替え**: デフォルト（基準順）／出版年の昇順・降順（欠損年は常に末尾）。
- **シリーズまとめ解除**: シリーズタブのときだけトグルを表示し、OFF で束を各巻へ
  展開して所蔵館数順に並べ直す。

### 検索（2 モード）

検索バーに**モード切替**を置く（`js/app.js` + `js/search.js`）。

- **収録データ検索（既定・サーバ不要）**: 事前生成した検索コーパス
  （`data/search/corpus-<類>.json`）を走査して NDC 類（1 桁）内を検索する。
  NDC 棚（`?ndc=<記号>`）表示中は**記号の 1 桁目の類**が範囲（例: `?ndc=913` は類 9）。
  既定棚では類セレクタで類を選ぶ。正規化（NFKC・英字大小・かな種・小書き）を
  かけて部分一致 AND、所蔵館数降順・最大 1,000 件。棚上部に**検索範囲**
  （収録データはスナップショットで CiNii 全体ではない旨）を明示する。
- **CiNii API 検索（サーバ稼働時のみ）**: `/api/search` を叩く現行経路。読込時の
  `/api/ping` で稼働判定し、**未稼働なら選択不可（グレーアウト＋理由表示）**。
- **1 文字語は不可**（両モード共通・「2 文字以上で入力してください」）。
- **空送信で棚（分類全体／既定データ）へ戻る**。
- 複数ファイル fetch の競合は **AbortController** で直前の検索を中断する。
- 正規化は Python（コーパス構築）と JS（クエリ側）で同一規則。**共有テストベクタ**
  （`core/search_normalize_vectors.json`）を双方のテストで検証する。

### URL 状態同期と初期条件（`?q=` / `?ndc=`）

- 検索語 `q`・NDC 分類 `ndc`（1〜3 桁）・モード `mode`・タブ `tab`・並べ替え `sort` を
  URL クエリへ反映し、共有・ブックマーク・戻る/進む（`history.pushState` / `popstate`）
  に対応。既定値は省略して最短の URL を保つ（`mode` は「ndc あり＋api」のときだけ書く）。
- URL 仕様（[docs/site-search.md](../docs/site-search.md)）:
  - `?ndc=<記号>&q=<語>` … 収録データ検索（範囲は記号 1 桁目の類）。空送信で `?ndc=<記号>` へ
  - `?ndc=<記号>&q=<語>&mode=api` … API の分類内検索
  - `?q=<語>&mode=api` / `?q=<語>`（mode なし）… API の全体検索（`?q=` のみは**後方互換**・
    同義。サーバ未稼働時は無言フォールバックをやめ、収録データ検索へ切り替える案内を出す）
- ページ読込時に URL から状態を復元し、モードに応じて検索を実行する。`ndc` の棚は
  静的 `data/ndc/<記号>.json` を読む（**サーバ不要**）。
- **NDC 棚の見出し**: 棚上部に分類記号・分類名・件数（総件数と収録件数）・
  データ取得日・出典（CiNii Books／JLA 公式 NDC 新訂9版データ〈NDC-LD・CC BY〉）を
  表示する。分類名・出典はマスタ `data/ndc/index.json` から引く。
- **NDC 棚内の検索**: 既定の収録データ検索は棚の**類（1 桁）**全体を対象に、事前生成
  コーパスを走査する（**サーバ不要**）。API 検索モードでは `/api/search?q=<語>&ndc=<記号>`
  を使う。空送信で分類全体の棚へ戻る。
  → 詳細は上の「検索（2 モード）」と [docs/site-search.md](../docs/site-search.md)。

### 詳細オーバーレイ

- 書影クリック（または Enter / Space）で表示。**PC は画面右パネル、モバイル
  （600px 以下）はボトムシート**で、下スワイプでも閉じられる。
- 表示項目は `books.json` にある項目のみ: タイトル・著者・出版社・出版年・
  所蔵館数・シリーズ（束の場合は所属巻の一覧）・ISBN・NCID・**CiNii へのリンク**。
- 分類・件名は詳細 API（`fetch`）整備後に追加する（後述）。

### レスポンシブ・アクセシビリティ

- 検索バーは主要 3 要素（タブ・並べ替え・検索窓）が 1 行に収まるかを**実測**して
  PC 1 行／モバイル 2 段組みを切り替える。
- `<meta name="viewport">`・相対単位・メディアクエリによるレスポンシブ設計。
- `role` / `aria-*` 属性・`:focus-visible` の可視フォーカス・
  `prefers-reduced-motion` 対応済み。

## 入力データ

- `build` の出力 `site/data/books.json`（`ownerCount` 降順）。スキーマは
  [`build/README.md`](../build/README.md) を参照。動的検索 API も同一スキーマを返す。
- 表紙 URL は各レコードの `coverUrl`（無ければ `null` → プレースホルダー表示）。

## 公開（GitHub Pages）

- `site/` 配下を Pages の公開ルートにする（`.github/workflows/pages.yml`）。
- すべて相対パスで参照し、サブパス公開（`/<repo>/`）でも動く。
- 動的検索を公開で使う場合は API を別ホストに置き、`js/app.js` の `SEARCH_URL` を
  絶対 URL へ差し替える。

### 7. パフォーマンス（P4〜P5）

- `books.json`（約 1.9 MB）は Pages の圧縮配信で当面問題ない。NDC 棚は 1 分類
  最大 10,000 件程度（約 4 MB・gzip 転送で約 700 KB）を想定しており、初期ロードの
  体感が悪ければ**分割ロード**（先頭 N 件＋残りの遅延読み込み）へ切り替える
  （`build` の分割出力と対。表示スキーマは変えない）。
- 描画済みカードが数千件になった場合の DOM 削減（仮想スクロール化）は、
  現状の逐次描画（PAGE_SIZE）で実害が出てから検討する。

### 8. キーワード検索棚の分類順ソート・分類見出し（P5・fetch 整備後）
