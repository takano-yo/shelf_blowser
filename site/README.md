# ③ site — 表示サイト（本棚 UI）

> **ステータス: 実装済み・公開中（GitHub Pages）**
> `build` が出力する `site/data/books.json` を初期表示し、検索時は API
> （`server/app.py` の `/api/search`）から**同一スキーマ**の books 配列を受け取って
> 本棚を丸ごと作り直す。バニラ JS・ビルド工程なし。
>
> 本書は「何が実装済みか（現状）」と「今後必要な作業（要件・手順）」を規定する。

---

## 目的

書誌データを **本（表紙）を横に並べた本棚** として表示し、図書館の書架
ブラウジング（棚を眺めて歩き、隣接する関連書に偶然出会う体験）を Web で再現する。

## 技術構成

- **静的サイト + バニラ JS**（ビルド工程なし）。HTML / CSS / JS のみ。
- `site/data/*.json` を `fetch()` で読み込み、表示・整列は**クライアントサイド**で行う。
- GitHub Pages にそのまま配置して公開できる（初期表示はサーバ不要）。
- 動的検索はバックエンド（`server/app.py`）の `/api/search` を利用する。
  API が無い・落ちている場合も初期の本棚は静的データで表示される
  （グレースフルデグレード）。

```
site/
├── README.md
├── index.html
├── css/styles.css
├── js/app.js          # データ読込・本棚描画・タブ/並べ替え・検索・オーバーレイ
└── data/              # build の出力（books.json / meta.json）
```

## 実装済みの機能（現状）

### 本棚ビュー

- **並び順**: `ownerCount`（所蔵館数）降順。主要書ほど棚の手前に並ぶ。
- ウィンドウ幅から列数を算出して明示グリッドに固定（最低 3 列）し、
  段ごとに「棚板」を描画する。
- 書影は書籍標準のアスペクト比で整列（実画像は `object-fit: cover`）。
- **プレースホルダー**（書影の無い書誌＝大多数・約 96%）: タイトル・著者を上部、
  出版社（シリーズものはシリーズ名）を下辺に描画。地色と下辺の文字色は
  **出版年に応じた連続グラデーション**（古＝暗い赤 → 中＝緑 → 新＝水色）で塗り、
  棚を一望したとき年代の分布が見えるようにしている。
- **無限スクロール**: 先頭 100 件（`PAGE_SIZE`）のみ描画し、棚末尾のセンチネルが
  近づくたびに次の 100 件を追加描画する。
- **ホバー詳細**: 表紙にマウスを重ねると、表紙と同サイズの詳細
  （著者・出版社・出版年・所蔵館数・シリーズ・ISBN）へフェードで交代する。

### シリーズ（親書誌）のまとめ

- `series[0].id`（親書誌 NCID）単位で束ね、**所蔵館数最大の 1 冊を代表表示**
  （棚での位置基準も代表の所蔵館数）。
- 「本が連なる」紙束の影・枠・冊数バッジで単独本と区別する。
- 多巻単独（ISBN 数 ≥ 4）も「シリーズ的」として同じ装飾の対象にする。

### タブ・並べ替え・シリーズまとめ解除

- **タブ**: すべて／単著・共著／編集書／シリーズ。`contribKind`（build 付与）と
  シリーズ的か否かで 3 バケットに分け、各タブは該当書籍のみを所蔵館数順に表示。
  「すべて」タブは**ブロック配分**（4〜8 冊のブロックへ編集書 1 冊・シリーズ 1 冊を
  混ぜる。固定シード乱数で並びは再現可能）。タブ間でスクロール位置を保存・復元する。
- **並べ替え**: デフォルト（基準順）／出版年の昇順・降順（欠損年は常に末尾）。
- **シリーズまとめ解除**: シリーズタブのときだけトグルを表示し、OFF で束を各巻へ
  展開して所蔵館数順に並べ直す。

### 検索（動的）

- 検索窓の送信で `/api/search?q=語` を fetch し、返った books 配列
  （`books.json` と同一スキーマ）で棚を作り直す。**空送信で既定データへ戻る**。
- 失敗時はメッセージを表示する（初期表示は静的データのため本棚自体は残る）。

### 詳細オーバーレイ

- 書影クリック（または Enter / Space）で表示。**PC は画面右パネル、モバイル
  （600px 以下）はボトムシート**で、下スワイプでも閉じられる。
- 表示項目は `books.json` にある項目のみ: タイトル・著者・出版社・出版年・
  所蔵館数・シリーズ（束の場合は所属巻の一覧）・ISBN・NCID・**CiNii へのリンク**。
- 分類・件名は詳細 API（`fetch`）整備後に追加する（後述）。

### レスポンシブ・アクセシビリティ

- 検索バーは主要 3 要素（タブ・並べ替え・検索窓）が 1 行に収まるかを**実測**して
  PC 1 行／モバイル 2 段組みを切り替える。
- `<meta name="viewport">`・相対単位・メディアクエリによるレスポンシブ設計。
- `role` / `aria-*` 属性・`:focus-visible` の可視フォーカス・
  `prefers-reduced-motion` 対応済み。

## 入力データ

- `build` の出力 `site/data/books.json`（`ownerCount` 降順）。スキーマは
  [`build/README.md`](../build/README.md) を参照。動的検索 API も同一スキーマを返す。
- 表紙 URL は各レコードの `coverUrl`（無ければ `null` → プレースホルダー表示）。

## 公開（GitHub Pages）

- `site/` 配下を Pages の公開ルートにする（`.github/workflows/pages.yml`）。
- すべて相対パスで参照し、サブパス公開（`/<repo>/`）でも動く。
- 動的検索を公開で使う場合は API を別ホストに置き、`js/app.js` の `SEARCH_URL` を
  絶対 URL へ差し替える。

## 今後必要な作業（要件・手順）

優先度は [ルート README のロードマップ](../README.md#今後必要な作業ロードマップ) に対応する。

### 1. URL 状態同期（P3）

- **目的**: 検索結果・表示状態を共有・ブックマークでき、ブラウザの戻る/進むが機能する
  ようにする（現状は URL に状態が乗らず、リロードで常に既定表示へ戻る）。
- **要件**:
  - 検索語 `q`・タブ `tab`・並べ替え `sort` を URL クエリへ反映する
    （例: `?q=漱石&tab=series&sort=year-asc`）。
  - ページ読込時に URL から状態を復元する（`q` があれば API を叩いて棚を再現）。
  - `history.pushState` / `popstate` で戻る・進むに追随する。
- **手順**: `app.js` の `runSearch` / `switchTab` / `changeSort` に URL 書き込みを追加し、
  `init()` に URL 読み取り → 状態復元を追加する。静的ホスティングのためパスでは
  なくクエリ文字列方式とする。

### 2. 検索リクエストの競合対策（P3）

- **目的**: 検索を連打したとき、遅れて返った古い応答が新しい結果を上書きする
  競合（レースコンディション）を防ぐ。
- **要件**: 新しい検索の開始時に進行中の fetch を `AbortController` で中断する。
  検索中はスピナー等の進行表示を出す（現状の「検索中…」表示は維持してよい）。
- **手順**: `runSearch` にモジュールレベルの `AbortController` を導入し、
  次の検索開始時に `abort()` → 新規作成して `fetch(url, { signal })` に渡す。
  `AbortError` はエラーメッセージを出さずに無視する。

### 3. ピボットブラウジング（P3）

- **目的**: Harvard StackLife の「クリックで文脈（棚）を切り替える」体験の導入。
  詳細オーバーレイの著者・出版社・シリーズ名から、その語の棚へ直接飛べるようにする。
- **要件**:
  - オーバーレイ内の著者・出版社・シリーズ名をクリック可能にし、クリックで
    その語を検索窓に入れて `runSearch(語)` を実行、オーバーレイを閉じて棚を作り直す。
  - URL 状態同期（#1）と連動する。
- **手順**: `overlayHtml()` の該当項目をボタン化 → クリックハンドラで検索を発火。
  まずは自由語検索として実装し、`fetch`／詳細索引の整備後に典拠 ID ベースの
  厳密な絞り込みへ置き換える。

### 4. 「棚の近傍」表示（P3）

- **目的**: Stanford SearchWorks の virtual browse のように、詳細オーバーレイにも
  「この本の棚での前後数冊」を出し、詳細画面からブラウジングを続けられるようにする。
- **要件**:
  - 現在の並び（`shelfItems`）上での位置から前後 N 冊（例: 各 5 冊）の小さな表紙列を
    オーバーレイ下部に表示する。
  - 近傍の表紙クリックでオーバーレイの内容をその本に差し替える（棚は動かさない）。
- **手順**: `openOverlay()` にアイテムの index を渡し、`shelfItems` から近傍を
  切り出して描画する。表紙描画は `coverHtml()` を縮小スタイルで再利用する。

### 5. アクセシビリティ強化（P3）

- **要件**:
  - タブ（`role="tablist"`）に WAI-ARIA Authoring Practices 準拠の
    **左右矢印キーでのタブ移動**を実装する。
  - 詳細オーバーレイ表示中の**フォーカストラップ**（Tab 移動がパネル外へ出ない）を
    実装する（現状はパネルへの初期フォーカスと復元のみ）。
  - ホバー詳細と同じ情報へキーボード・タッチでも到達できることを確認する
    （詳細オーバーレイで代替済みだが、導線をドキュメント化する）。

### 6. パフォーマンス（P3〜P4）

- `books.json`（約 1.9 MB）は Pages の圧縮配信で当面問題ないが、より件数の多い
  分野へ広げる場合は**分割ロード**（先頭 N 件＋残りの遅延読み込み）へ切り替える
  （`build` の分割出力と対。表示スキーマは変えない）。
- 描画済みカードが数千件になった場合の DOM 削減（仮想スクロール化）は、
  現状の逐次描画（PAGE_SIZE）で実害が出てから検討する。

### 7. NDC 分類順の棚（P4・fetch 整備後）

- **目的**: 「請求記号順に棚を眺める」という書架ブラウジングの本来形の再現。
  現状の棚は所蔵館数順＝「主要書から眺める棚」だが、分類記号（NDC）順の棚は
  「主題の隣接を歩く棚」になる（SearchWorks の virtual browse は請求記号順）。
- **要件**:
  - `fetch` が取得する詳細レコードの NDC（`dc:subject`）を `build` で各レコードへ
    付与する（→ [build/README.md](../build/README.md) の詳細索引）。
  - 並べ替えに「分類順」を追加し、NDC 昇順（同分類内は所蔵館数降順）で並べる。
  - 棚の途中に分類見出し（例: `910.26 日本文学—近代`）を差し込み、
    いま棚のどこに居るかを示す。
- **依存**: fetch の実装（[fetch/README.md](../fetch/README.md)）→ build の詳細索引生成。

### 8. ファセット検索 UI（P4）

- `books.json` の項目から作れる**暫定ファセット**（出版年代・出版者・シリーズ）を
  検索窓まわりに置き、選択で表示中の棚を絞り込めるようにする。
- 件名・典拠著者・分類のファセットは `fetch`／`build` の詳細索引整備後に追加する。
